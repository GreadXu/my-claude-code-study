# Learning Bookmark Skill

## 功能范围
- 创建书签（自动获取上下文、智能生成目标）
- 继续书签（进入探索模式）
- 完成书签（返回主线学习）
- 删除书签

---

## 创建书签

**触发方式**（任意一种）：
- "创建书签 `<名称>`" / "添加书签 `<名称>`" / "新建书签 `<名称>`"
- "记个书签 `<名称>`" / "添加个书签 `<名称>`" / "做个书签 `<名称>`"
- "创建书签" / "添加书签" / "新建书签"（不指定名称时询问）

**执行步骤**：

### 步骤 1：自动获取上下文
- 从 `PROGRESS.md` 获取当前"进行中"的模块
- 从模块的 `checklist.md` 获取当前学习位置（最后一个有勾选的章节）

### 步骤 2：询问用户
- 疑问点描述（想要探索什么问题？）
- 探索目标（可选，系统可智能生成）

### 步骤 3：智能生成内容
根据疑问点类型，自动生成探索目标和返回验证：

| 疑问类型 | 探索目标 | 返回验证 |
|----------|----------|----------|
| 命令相关 | 阅读文档、实践使用、记录技巧 | 熟练使用该命令 |
| 概念相关 | 阅读资料、整理笔记 | 能够解释概念 |
| 配置相关 | 查看文档、实践配置 | 能够独立配置 |
| 机制相关 | 阅读源码、实验验证 | 能够说明工作原理 |

### 步骤 4：显示预览
```
╔═══════════════════════════════════════════════════════╗
║                    书签预览                           ║
╠═══════════════════════════════════════════════════════╣
║ 名称：深入理解 Tool Use 机制                          ║
║ 来源模块：ai-tools-fundamentals                       ║
║ 主线位置：完整模式 > 第一部分 > 架构与概念            ║
╠═══════════════════════════════════════════════════════╣
║ 疑问点：                                              ║
║   Tool Use 的具体实现原理是什么？                      ║
╠═══════════════════════════════════════════════════════╣
║ 探索目标：                                            ║
║   - [ ] 阅读官方 Tool Use 文档                        ║
║   - [ ] 查看 Tool Use 的源码实现                      ║
╠═══════════════════════════════════════════════════════╣
║ 返回验证：                                            ║
║   - [ ] 能够解释 Tool Use 的工作原理                  ║
╚═══════════════════════════════════════════════════════╝
```

### 步骤 5：确认并保存
用户确认后，将书签添加到 `LEARNING_BOOKMARKS.md`

## 继续书签

**触发方式**（任意一种）：
- "继续书签" / "查看书签" / "我的书签" / "书签列表"
- "看看书签" / "书签情况" / "显示书签"
- "回到书签" / "继续探索" / "探索书签"
- "继续书签 `<名称>`" / "查看书签 `<名称>`"（直接继续指定书签）

**执行步骤**：

1. 读取 `LEARNING_BOOKMARKS.md`

2. 如果未指定书签名称：
   - 显示所有 🔵 探索中 和 🟡 待探索 的书签列表
   - 等待用户选择

3. 如果指定了书签名称：
   - 直接定位到该书签

4. 显示书签详情和探索笔记

5. 更新书签状态为 🔵 探索中

6. ⭐ **显示探索模式提示**：
   ```
   ════════════════════════════════════════════════════════
   🔵 书签探索模式已启动

   当前书签：{书签名称}
   疑问点：{疑问点描述}

   💡 提示：
   • 探索期间请专注于当前疑问
   • 完成后说"完成书签"返回主线
   ════════════════════════════════════════════════════════
   ```

7. 询问用户下一步：
   - 开始探索？
   - 查看探索笔记？
   - 修改探索目标？

## 书签探索行为规范

> **当书签状态为 🔵 探索中 时，AI 应遵循以下行为规范：**

### 探索阶段

1. **每次查询/操作完成后**：
   - 显示探索结果摘要（简洁，不超过 5 个要点）
   - ⭐ **关键：主动询问** "还有其他疑问吗？"
   - 等待用户响应，不要自动继续或返回主线

2. **用户回应的处理**：
   - 用户提出新疑问 → 继续探索，记录到探索笔记
   - 用户说"没有了"/"没问题" → 提示："可以说'完成书签'返回主线"
   - 用户说"完成书签" → 调用完成书签流程
   - 用户沉默/不明确 → 保持探索中状态，等待用户主动

3. **输出格式示例**：
   ```
   📚 探索结果：Tool Use 机制

   核心要点：
   • Tool Use 通过 function calling 实现
   • AI 主动调用外部工具，而非被动响应
   • 支持多工具并行调用

   ❓ 还有其他疑问吗？
      - 有疑问 → 直接告诉我
      - 没有了 → 说"完成书签"返回主线
   ```

### 禁止行为

- ❌ 不要在查询完成后自动说"让我们返回主线学习"
- ❌ 不要推测用户没有疑问
- ❌ 不要在没有用户确认的情况下结束探索

### 探索笔记记录（可选）

1. **触发条件**：用户明确说以下任一短语时触发
   - "记录一下"
   - "记到笔记里"
   - "保存这个"
   - "写到书签笔记"

2. **记录内容**：
   - 当前探索结果的要点摘要（与显示给用户的内容一致）
   - 记录时间戳

3. **记录位置**：
   - 添加到 `LEARNING_BOOKMARKS.md` 对应书签的"探索笔记"区域

4. **记录格式示例**：
   ```markdown
   ## 探索笔记

   ### 2026-03-07

   **Tool Use 机制探索**：

   - Tool Use 通过 function calling 实现
   - AI 主动调用外部工具，而非被动响应
   - 支持多工具并行调用
   ```

5. **记录后提示**：
   ```
   ✅ 已记录到书签笔记
   ```

6. **非自动记录**：
   - 用户未明确要求时，不自动记录
   - 避免文件频繁修改

## 完成书签

**触发方式**（任意一种）：
- "完成书签 `<名称>`" / "标记书签 `<名称>` 为完成" / "书签完成了 `<名称>`"
- "书签搞定 `<名称>`" / "书签学完了 `<名称>`" / "结束书签 `<名称>`"
- "完成这个书签" / "书签已完成" / "书签做完了"

**执行步骤**：

1. 显示书签详情和返回验证清单

2. 询问确认：
   - 返回验证是否都已满足？
   - 确认标记为完成？

3. 用户确认后：
   - 标记书签为 🟢 已完成
   - 记录完成时间（YYYY-MM-DD）
   - 更新 `LEARNING_BOOKMARKS.md` 中的统计

4. ⭐ **显示主线返回信息**：
   ```
   ════════════════════════════════════════════════════════
   ✅ 书签已完成，正在返回主线学习...

   📍 主线位置：
      模块：{模块名}
      进度：{学习模式} > {章节}
      下一步：{下一学习项}

   继续主线学习？(y/n)
   ════════════════════════════════════════════════════════
   ```

## 删除书签

**触发方式**（任意一种）：
- "删除书签 `<名称>`" / "移除书签 `<名称>`" / "去掉书签 `<名称>`"
- "删掉书签 `<名称>`" / "书签 `<名称>` 不要了" / "清除书签 `<名称>`"

**执行步骤**：

1. 显示警告：此操作将删除书签及其所有探索笔记

2. 询问确认（需要输入书签名称确认）

3. 从 `LEARNING_BOOKMARKS.md` 中移除书签

4. 更新统计信息

## 注意事项

### 书签探索期间
- AI 应专注于书签疑问，避免讨论主线内容
- 探索结果使用要点列表，保持简洁（避免占用过多上下文）
- 每次查询后必须主动询问"还有其他疑问吗？"

### 返回主线时
- 显示主线位置和下一步，帮助用户重新定位
- 避免延续书签话题，专注主线学习内容

### 日期格式
统一使用 YYYY-MM-DD 格式

### 状态标识
- 🟡 待探索
- 🔵 探索中
- 🟢 已完成
