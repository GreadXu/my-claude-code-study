## 📝 我的学习笔记

### 2026-03-01

今天学习了 `/edit` 命令（实际是通过 Edit 工具），掌握了以下要点：

1. **必须先 Read 文件**：编辑前必须先读取文件获取当前状态
2. **精确匹配**：old_string 必须与原文完全一致，包括空格和缩进
3. **唯一性要求**：如果匹配到多处，需要使用 replace_all 或提供更多上下文
4. **三种替换模式**：
   - 简单单行替换
   - 多行文本替换
   - 全局替换（replace_all: true）

### 使用技巧总结

| 场景 | 方法 |
|------|------|
| 文本唯一 | 直接替换 |
| 文本重复且需全部替换 | 使用 `replace_all: true` |
| 文本重复但只需替换一处 | 提供更多上下文 |
| 多行文本 | 保持原有缩进格式 |

### 与传统命令对比

| 特性 | Edit 工具 | sed/awk |
|------|-----------|---------|
| 安全性 | 高（精确匹配） | 低（正则可能误匹配） |
| 可读性 | 高（清晰明确） | 低（需理解正则） |
| 学习曲线 | 低 | 高 |
| 适用场景 | 结构化文件编辑 | 流式文本处理 |
