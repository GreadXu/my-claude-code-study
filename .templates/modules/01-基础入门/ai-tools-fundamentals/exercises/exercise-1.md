## /edit 命令详解

### 语法
```markdown
/edit <文件路径>
```

### 参数说明
| 参数 | 说明 | 必需 |
|------|------|------|
| `old_string` | 要替换的原始文本 | ✅ |
| `new_string` | 替换后的新文本 | ✅ |
| `replace_all` | 是否全局替换所有匹配项 | ❌ |

### 关键规则
1. ✅ **必须先 Read**：编辑前必须先读取文件
2. ✅ **精确匹配**：`old_string` 必须与原文完全一致（包括空格、缩进）
3. ✅ **唯一性**：`old_string` 应该唯一，否则需要使用 `replace_all`
4. ✅ **使用 Edit 工具**：通过 Edit 工具执行编辑，而非 `/edit` 斜杠命令

## 常见错误

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| `old_string is not unique` | 匹配到多处 | 使用 `replace_all: true` |
| `Edit without reading file` | 未先读取 | 先使用 Read 工具 |
| `No match found` | 文本不匹配 | 检查空格、缩进是否完全一致 |
