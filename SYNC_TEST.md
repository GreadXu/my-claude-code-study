# 同步功能测试指南

## 测试步骤

### 团队成员操作：

1. **检查更新**
   ```
   /study sync
   ```

2. **执行同步**
   ```
   /study sync auto
   ```

3. **验证结果**
   - 查看版本号是否更新到 1.3.1
   - 确认个人学习进度未被覆盖
   - 检查新增的 SYNC_TEST.md 文件

### 预期输出：

```
╔════════════════════════════════════════════════════════╗
║              准备同步上游更新                        ║
╚════════════════════════════════════════════════════════╝

当前版本: 1.3.0
上游版本: 1.3.1
```

---

## 关键文件

| 文件 | 操作 |
|------|------|
| `CHANGELOG.md` | 更新版本号 1.3.0 → 1.3.1 |
| `SYNC_TEST.md` | 新建测试指南 |

## 验证步骤

### 维护者（你）操作：

1. **提交更新**
   ```bash
   git add CHANGELOG.md SYNC_TEST.md
   git commit -m "test: 版本 1.3.1 - 同步功能测试"
   git push origin main
   ```

2. **通知团队成员**
   - 告知版本已更新到 1.3.1
   - 让他们执行任何 `/study` 命令测试自动提醒

### 团队成员验证（自动化测试）：

**步骤 1**：执行任意学习命令（会自动触发更新检查）
```
/study start claude-code-core
# 或
/study update
# 或
/study complete
```

**预期输出**：命令输出前会显示更新提醒框
```
╔═══════════════════════════════════════════════════════════════════╗
║ ⚠ 发现学习计划更新可用                                          ║
╠═══════════════════════════════════════════════════════════════════╣
║ 当前版本: 1.3.0                                                 ║
║ 上游版本: 1.3.1                                                 ║
╠═══════════════════════════════════════════════════════════════════╣
║ 建议：学习结束后运行 "同步学习计划" 或 /study sync auto        ║
╚═══════════════════════════════════════════════════════════════════╝

[原命令输出继续...]
```

**步骤 2**：执行同步
```
/study sync auto
```

**步骤 3**：验证结果
- ✅ 版本号显示为 1.3.1（再次运行 `/study` 命令不再显示提醒）
- ✅ SYNC_TEST.md 文件存在
- ✅ 个人学习进度（PROGRESS.md、checklist.md）完好

---

## 回滚方案

如果测试出现问题：

```bash
# 回退到测试前版本
git revert <commit-hash>
git push origin main

# 或直接创建新版本修复
git commit -m "fix: 回滚同步测试版本"
```
