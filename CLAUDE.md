# Claude Code 学习计划 - AI 工作流程指南

## 学习工作流程命令

当用户请求以下操作时，按照对应模式执行。**支持自然语言触发**。

### 开始学习

**触发方式**（任意一种）：
- "开始学习 `<模块名>`"
- "启动 `<模块名>` 的学习"
- "开始学习模块 `<模块名>`"

**执行步骤**：

1. 查找模块路径（从以下映射）：
   - `claude-code-core` → `01-基础入门/claude-code-core`
   - `mcp-basics` → `01-基础入门/mcp-basics`
   - `agent-sdk` → `02-进阶探索/agent-sdk`
   - `mcp-advanced` → `02-进阶探索/mcp-advanced`
   - `everything-claude-code` → `02-进阶探索/everything-claude-code`
   - `cc-switch` → `03-实战应用/cc-switch`
   - `spec-kit` → `03-实战应用/spec-kit`
   - `projects` → `03-实战应用/projects`

2. 验证模块是否已开始学习（检查 checklist.md 的学习模式选择）
   - 如果已开始，显示警告信息

3. 询问用户选择学习模式：
   - 1. 📚 快速模式（约 3 天）- 理论为主
   - 2. 🛠️ 完整模式（约 1 周）- 理论+实践

4. 显示更新预览，包括：
   - 模块信息（名称、优先级、模式、开始日期）
   - 将要更新的文件列表
   - 等待用户确认

5. 用户确认后，更新以下文件：
   - **checklist.md**：勾选对应的学习模式 checkbox
   - **notes.md**：设置学习模式和开始日期
   - **PROGRESS.md**：更新模块状态为"进行中"，添加学习日志

6. 显示成功信息和下一步指导

### 更新进度

**触发方式**：
- "更新进度 `<模块名>`"
- "更新 `<模块名>` 的学习进度"
- "同步进度 `<模块名>`"

**执行步骤**：

1. 读取模块的 `checklist.md`

2. 解析学习模式和完成项：
   - 检测已勾选的学习模式（快速/完整）
   - 统计已完成的 checkbox 数量
   - 根据学习模式计算对应的完成百分比

3. 读取 `PROGRESS.md` 获取当前进度

4. 生成更新预览：
   ```
   Progress: X% -> Y%
   Status: 进行中
   进度条: ██████░░░░░░░░░░░░░░ 30%
   ```

5. 用户确认后更新 `PROGRESS.md`：
   - 更新模块的完成百分比
   - 添加学习日志条目

**进度计算规则**：
- 快速模式：只统计"📚 快速模式"章节下的 checkbox
- 完整模式：统计"🛠️ 完整模式"章节下的所有 checkbox
- 未选择模式：基于完整模式计算

### 完成学习

**触发方式**：
- "完成学习 `<模块名>`"
- "标记 `<模块名>` 为已完成"
- "完成模块 `<模块名>`"

**执行步骤**：

1. 解析 `checklist.md` 计算完成百分比

2. 验证完成情况：
   - 如果完成度 < 100%，显示警告和未完成项列表
   - 询问是否确认完成
   - 如果用户坚持，可以强制完成（即使有未完成项）

3. 显示完成预览：
   - 最终完成百分比
   - 完成日期
   - 将要更新的文件

4. 用户确认后，更新以下文件：
   - **notes.md**：记录完成日期
   - **PROGRESS.md**：更新状态为"已完成"，进度设为 100%，添加完成日志

5. 重新计算总体进度和各优先级进度

### 查看状态

**触发方式**：
- "查看学习状态"
- "显示进度概览"
- "学习进度怎么样了"
- "查看 `<模块名>` 的状态"（指定模块）

### 新增模块

**触发方式**：
- "新增模块 `<模块名>`"
- "添加模块 `<模块名>`"
- "创建新模块 `<模块名>`"

**执行步骤**：

1. 询问用户确认模块信息：
   - 模块名称（英文短名称，如 `my-new-module`）
   - 所属阶段（01-基础入门/02-进阶探索/03-实战应用）
   - 优先级（P0 必学/P1 推荐/P2 可选/P3 了解）
   - 预计时长（完整模式）
   - 模式支持（快速/完整/两者）

2. 显示预览，等待用户确认

3. 创建模块目录和文件：
   - 创建 `XX-阶段名称/模块名/` 目录
   - 生成 `README.md`（模块导学）
   - 生成 `checklist.md`（学习清单模板）
   - 生成 `notes.md`（学习笔记模板）

4. 更新以下文件：
   - **PROGRESS.md**：添加模块行到对应阶段的表格
   - **CLAUDE.md**：添加模块路径映射到"开始学习"章节

5. 添加系统日志

---

### 更新模块信息

**触发方式**：
- "更新模块信息 `<模块名>`"
- "修改模块 `<模块名>`"
- "编辑模块 `<模块名>`"

**执行步骤**：

1. 读取模块当前信息

2. 询问用户要修改的字段：
   - 名称
   - 优先级
   - 预计时长
   - 模式支持

3. 显示变更预览

4. 更新相关文件：
   - `XX-阶段名称/模块名/README.md`
   - `PROGRESS.md`
   - `CLAUDE.md`（如名称或路径变更）

---

### 重置模块

**触发方式**：
- "重置模块 `<模块名>`"
- "重新开始 `<模块名>`"
- "清除 `<模块名>` 的进度"

**执行步骤**：

1. 显示警告：此操作将清除模块的所有学习进度

2. 询问用户确认

3. 重置以下文件：
   - **checklist.md**：取消所有勾选项，清除学习模式选择
   - **notes.md**：重置日期和内容
   - **PROGRESS.md**：状态改为"未开始"，进度改为 0%

4. 添加重置日志

---

### 删除模块

**触发方式**：
- "删除模块 `<模块名>`"
- "移除模块 `<模块名>`"

**执行步骤**：

1. 显示警告：此操作将删除模块目录和所有相关文件

2. 询问用户确认（需要输入模块名确认）

3. 执行删除：
   - 删除模块目录及其所有文件
   - 从 PROGRESS.md 移除模块行

## 文件操作模式

### checklist.md 更新模式

**设置学习模式**：
```markdown
# 修改前
- [ ] **快速模式**（约 3 天）- 理论为主
- [x] **完整模式**（约 1 周）- 理论+实践

# 修改为（快速模式）
- [x] **快速模式**（约 3 天）- 理论为主
- [ ] **完整模式**（约 1 周）- 理论+实践
```

### notes.md 更新模式

```markdown
# 修改前
> **学习模式**：待选择
> **开始日期**：待定

# 修改为（开始快速模式学习）
> **学习模式**：快速模式
> **开始日期**：2026-02-26
```

### PROGRESS.md 更新模式

```markdown
# 修改模块行
# 从：
| claude-code-core | P0 🔴 | 未选择 | 未开始 | 0% |

# 改为（开始快速模式）：
| claude-code-core | P0 🔴 | 快速模式 | 进行中 | 0% |

# 添加学习日志
| 2026-02-26 | claude-code-core | 开始学习 | 选择快速模式 |
```

## 注意事项

1. **用户确认**：修改多个文件前，总是先显示预览并等待用户确认
2. **日期格式**：统一使用 YYYY-MM-DD 格式
3. **文件路径**：所有路径相对于仓库根目录
4. **错误处理**：如果文件不存在或模块未找到，给出明确的错误提示
5. **进度计算**：基于实际勾选的 checkbox 数量，不要估算
