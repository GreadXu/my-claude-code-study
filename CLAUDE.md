# AI 学习系统 - 工作流程指南

## 项目概述

这是一个 **通用化的 AI 驱动学习框架**，支持学习任何技术领域。通过学习目标导向、AI 引导创建课程、智能缓存管理，实现个性化学习体验。

### v4.0 新特性
- 📚 **多学习目标管理** - 同时管理多个学习目标，支持切换、暂停、归档
- 🤖 **AI 引导创建** - 交互式创建个性化学习目标，自动生成课程结构
- 📢 **缓存更新提示** - 刷新缓存时自动检测课程变更，生成修改建议
- 🎯 **目标导向学习** - 以学习目标为核心，整合课程、进度、书签、笔记

```

│  课程层（Courses）                                       │
│  • 课程定义与结构                                        │

│  • 知识缓存关联                                          │

│  数据层（Data）                                          │
│  • 学习进度（.learning/progress/）                      │
│  • 书签（.learning/bookmarks/）                         │
│  • 笔记（.learning/notes/）                             │
```

## 数据隔离原则

### 系统内容 vs 个人数据

| 类型 | 位置 | Git 追踪 | 同步影响 |
|------|------|----------|----------|
| 系统模板 | `.templates/modules/` | ✅ 是 | 可更新 |
| Skill 定义 | `.claude/skills/` | ✅ 是 | 可更新 |
| 学习目标 | `.learning/goals/` | ❌ 否 | 不受影响 |
| 课程定义 | `.learning/courses/` | ❌ 否 | 不受影响 |
| 个人进度 | `.learning/progress/` | ❌ 否 | 不受影响 |
| 书签笔记 | `.learning/bookmarks/,notes/` | ❌ 否 | 不受影响 |
| 知识缓存 | `.learning/cache/` | ❌ 否 | 不受影响 |

## 从 v3.0 迁移

如果您是 v3.0 用户，运行迁移脚本：

```bash
# 预演模式（推荐先运行）
bash scripts/migrate-v4.sh --dry-run

# 正式迁移
bash scripts/migrate-v4.sh

# 自动迁移（使用默认配置）
bash scripts/migrate-v4.sh --auto
```

迁移脚本将：
1. 自动备份现有数据到 `.backups/migrate-v4-*/`
2. 创建新的 `.learning/` 目录结构
3. 迁移进度、书签、笔记、缓存到目标系统
4. 创建默认学习目标和课程定义
5. 验证迁移结果

## 常见问题

### 如何开始学习？

**方式 1：使用现有模块（v3.0 兼容）**
```
说 "开始学习 <模块名>"
```

**方式 2：创建个性化学习目标（推荐）**
```
说 "创建学习目标" 或 "我要学 X"
AI 将引导您创建个性化目标并生成课程
```

### 如何查看当前进度？

```
说 "查看学习状态" 查看所有模块进度
说 "当前目标" 查看激活的学习目标
说 "查看目标" 查看所有目标概览
```

### 如何切换学习目标？

```
说 "切换目标 <目标名>"
系统会自动保存当前目标进度并加载新目标
```

### 缓存更新提示是什么？

当您说 "刷新缓存" 时，系统会：
1. 拉取最新资料
2. 对比新旧版本，识别变更
3. 生成课程修改建议（新增/更新/删除）
4. 询问是否应用变更

## 技术架构

本系统采用 **学习目标导向架构**：

```
学习目标 (Goal)
    ↓ 包含
课程 (Course)
    ↓ 包含
模块 (Module)
    ↓ 包含
学习项 (Learning Item)
```

**数据流**：
```
用户输入 → Skill 处理 → 数据更新 → 文件存储 → 状态反馈
```

**缓存关联**：
```
课程缓存 (.learning/cache/courses/)
    ↓ 引用
共享缓存 (.learning/cache/shared/)
```
