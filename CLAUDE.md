# Claude Code 学习计划 - AI 工作流程指南

## 学习工作流程命令

当用户请求以下操作时，按照对应模式执行。**支持自然语言触发**。

### 开始学习

**触发方式**（任意一种）：
- "开始学习 `<模块名>`"
- "启动 `<模块名>` 的学习"
- "开始学习模块 `<模块名>`"

**执行步骤**：

0. **检查知识缓存**：
   - 读取模块的 `knowledge/.metadata.json`（如果存在）
   - **情况 A：缓存存在**
     - 显示：`发现本地缓存 (缓存日期: YYYY-MM-DD)`
     - 询问用户：
       - 1. 使用本地缓存（快速开始）
       - 2. 更新缓存（拉取最新资料）
   - **情况 B：缓存不存在**
     - 显示：`未找到知识缓存`
     - 询问用户：
       - 1. 初始化缓存（推荐，首次拉取）
       - 2. 跳过缓存（直接开始学习）

1. 查找模块路径（从以下映射）：
   - `claude-code-core` → `01-基础入门/claude-code-core`
   - `mcp-basics` → `01-基础入门/mcp-basics`
   - `agent-sdk` → `02-进阶探索/agent-sdk`
   - `mcp-advanced` → `02-进阶探索/mcp-advanced`
   - `openclaw-ecosystem` → `02-进阶探索/openclaw-ecosystem`
   - `everything-claude-code` → `02-进阶探索/everything-claude-code`
   - `cc-switch` → `03-实战应用/cc-switch`
   - `spec-kit` → `03-实战应用/spec-kit`
   - `projects` → `03-实战应用/projects`

2. 验证模块是否已开始学习（检查 checklist.md 的学习模式选择）
   - 如果已开始，显示警告信息

3. 询问用户选择学习模式：
   - 1. 📚 快速模式（约 3 天）- 理论为主
   - 2. 🛠️ 完整模式（约 1 周）- 理论+实践

4. 显示更新预览，包括：
   - 模块信息（名称、优先级、模式、开始日期）
   - 将要更新的文件列表
   - 等待用户确认

5. 用户确认后，更新以下文件：
   - **checklist.md**：勾选对应的学习模式 checkbox
   - **notes.md**：设置学习模式和开始日期
   - **PROGRESS.md**：更新模块状态为"进行中"，添加学习日志

6. 显示成功信息和下一步指导

### 更新进度

**触发方式**：
- "更新进度 `<模块名>`"
- "更新 `<模块名>` 的学习进度"
- "同步进度 `<模块名>`"

**执行步骤**：

1. 读取模块的 `checklist.md`

2. 解析学习模式和完成项：
   - 检测已勾选的学习模式（快速/完整）
   - 统计已完成的 checkbox 数量
   - 根据学习模式计算对应的完成百分比

3. 读取 `PROGRESS.md` 获取当前进度

4. 生成更新预览：
   ```
   Progress: X% -> Y%
   Status: 进行中
   进度条: ██████░░░░░░░░░░░░░░ 30%
   ```

5. 用户确认后更新 `PROGRESS.md`：
   - 更新模块的完成百分比
   - 添加学习日志条目

**进度计算规则**：
- 快速模式：只统计"📚 快速模式"章节下的 checkbox
- 完整模式：统计"🛠️ 完整模式"章节下的所有 checkbox
- 未选择模式：基于完整模式计算

### 完成学习

**触发方式**：
- "完成学习 `<模块名>`"
- "标记 `<模块名>` 为已完成"
- "完成模块 `<模块名>`"

**执行步骤**：

1. 解析 `checklist.md` 计算完成百分比

2. 验证完成情况：
   - 如果完成度 < 100%，显示警告和未完成项列表
   - 询问是否确认完成
   - 如果用户坚持，可以强制完成（即使有未完成项）

3. 显示完成预览：
   - 最终完成百分比
   - 完成日期
   - 将要更新的文件

4. 用户确认后，更新以下文件：
   - **notes.md**：记录完成日期
   - **PROGRESS.md**：更新状态为"已完成"，进度设为 100%，添加完成日志

5. 重新计算总体进度和各优先级进度

### 新增模块

**触发方式**：
- "新增模块 `<模块名>`"
- "添加模块 `<模块名>`"
- "创建新模块 `<模块名>`"

**执行步骤**：

1. 询问用户确认模块信息：
   - 模块名称（英文短名称，如 `my-new-module`）
   - 所属阶段（01-基础入门/02-进阶探索/03-实战应用）
   - 优先级（P0 必学/P1 推荐/P2 可选/P3 了解）
   - 预计时长（完整模式）
   - 模式支持（快速/完整/两者）

2. 显示预览，等待用户确认

3. 创建模块目录和文件：
   - 创建 `XX-阶段名称/模块名/` 目录
   - 生成 `README.md`（模块导学）
   - 生成 `checklist.md`（学习清单模板）
   - 生成 `notes.md`（学习笔记模板）

4. 更新以下文件：
   - **PROGRESS.md**：添加模块行到对应阶段的表格
   - **CLAUDE.md**：添加模块路径映射到"开始学习"章节

5. 添加系统日志

## 文件操作模式

### checklist.md 更新模式

**设置学习模式**：
```markdown
# 修改前
- [ ] **快速模式**（约 3 天）- 理论为主
- [x] **完整模式**（约 1 周）- 理论+实践

# 修改为（快速模式）
- [x] **快速模式**（约 3 天）- 理论为主
- [ ] **完整模式**（约 1 周）- 理论+实践
```

### notes.md 更新模式

```markdown
# 修改前
> **学习模式**：待选择
> **开始日期**：待定

# 修改为（开始快速模式学习）
> **学习模式**：快速模式
> **开始日期**：2026-02-26
```

### PROGRESS.md 更新模式

```markdown
# 修改模块行
# 从：
| claude-code-core | P0 🔴 | 未选择 | 未开始 | 0% |

# 改为（开始快速模式）：
| claude-code-core | P0 🔴 | 快速模式 | 进行中 | 0% |

# 添加学习日志
| 2026-02-26 | claude-code-core | 开始学习 | 选择快速模式 |
```

## 学习书签命令

> 用于追踪学习过程中的疑问分支和深入学习点，确保探索后能返回主线。

### 创建书签

**触发方式**（任意一种）：
- "创建书签 `<名称>`"
- "添加书签 `<名称>`"
- "新建书签 `<名称>`"

**执行步骤**：

1. **自动获取上下文**：
   - 从 `PROGRESS.md` 获取当前"进行中"的模块
   - 从模块的 `checklist.md` 获取当前学习位置（最后一个有勾选的章节）

2. **询问用户**：
   - 疑问点描述（想要探索什么问题？）
   - 探索目标（可选，系统可智能生成）

3. **智能生成内容**：
   根据疑问点类型，自动生成探索目标和返回验证：

   | 疑问类型 | 探索目标 | 返回验证 |
   |----------|----------|----------|
   | 命令相关 | 阅读文档、实践使用、记录技巧 | 熟练使用该命令 |
   | 概念相关 | 阅读资料、整理笔记 | 能够解释概念 |
   | 配置相关 | 查看文档、实践配置 | 能够独立配置 |
   | 机制相关 | 阅读源码、实验验证 | 能够说明工作原理 |

4. **显示预览**：
   ```
   ╔═══════════════════════════════════════════════════════╗
   ║                    书签预览                           ║
   ╠═══════════════════════════════════════════════════════╣
   ║ 名称：深入理解 Tool Use 机制                          ║
   ║ 来源模块：claude-code-core                            ║
   ║ 主线位置：完整模式 > 第一部分 > 架构与概念            ║
   ╠═══════════════════════════════════════════════════════╣
   ║ 疑问点：                                              ║
   ║   Tool Use 的具体实现原理是什么？                      ║
   ╠═══════════════════════════════════════════════════════╣
   ║ 探索目标：                                            ║
   ║   - [ ] 阅读官方 Tool Use 文档                        ║
   ║   - [ ] 查看 Tool Use 的源码实现                      ║
   ╠═══════════════════════════════════════════════════════╣
   ║ 返回验证：                                            ║
   ║   - [ ] 能够解释 Tool Use 的工作原理                  ║
   ╚═══════════════════════════════════════════════════════╝
   ```

5. 用户确认后，将书签添加到 `LEARNING_BOOKMARKS.md`

- "继续书签"

- "我的书签"
- "书签列表"

**执行步骤**：

1. 读取 `LEARNING_BOOKMARKS.md`

2. 如果未指定书签名称：
   - 显示所有 🔵 探索中 和 🟡 待探索 的书签列表
   - 等待用户选择

3. 如果指定了书签名称：
   - 直接定位到该书签

4. 显示书签详情和探索笔记

5. 更新书签状态为 🔵 探索中

6. 询问用户下一步：
   - 开始探索？
   - 查看探索笔记？
   - 修改探索目标？

### 初始化知识缓存

**触发方式**（任意一种）：
- "初始化知识缓存 `<模块名>`"
- "创建知识缓存 `<模块名>`"

**执行步骤**：

1. 从配置获取模块资料来源

2. 拉取官方资料（使用 MCP 工具，优化参数：`retain_images=false`，`timeout=20`）

3. 创建模块知识缓存目录：
   ```
   XX-阶段名称/模块名/knowledge/
   ├── README.md           # 缓存说明
   ├── overview.md         # 概述内容
   ├── concepts.md         # 核心概念
   ├── guides.md           # 使用指南
   └── .metadata.json      # 缓存元数据
   ```

4. 结构化存储到缓存文件

5. 更新 `KNOWLEDGE_CACHE.md` 状态表

**优化参数说明**：
- `retain_images=false`：跳过图片下载，提升加载速度 50-70%
- `timeout=20`：20秒超时，适合大型文档

### 刷新知识缓存

**触发方式**（任意一种）：
- "刷新知识缓存"
- "更新知识缓存"
- "刷新知识缓存 `<模块名>`"

1. 显示当前缓存状态（从 `KNOWLEDGE_CACHE.md`）

2. 询问更新范围：
   - 1. 全部模块
   - 2. 指定模块

3. 拉取最新资料（使用优化参数：`retain_images=false`，`timeout=20`）

4. 更新缓存文件和 `.metadata.json`

5. 更新 `KNOWLEDGE_CACHE.md` 更新历史

**优化参数说明**：
- `retain_images=false`：跳过图片下载，提升加载速度 50-70%
- `timeout=20`：20秒超时，适合大型文档

### 查看知识缓存

## "开始学习"流程更新（集成缓存检查）

在原"开始学习"流程的步骤 1 之前，插入缓存检查：

```
0. 检查知识缓存
   └─ 读取模块的 knowledge/.metadata.json

   情况 A：缓存存在
   ├─ 显示：发现本地缓存 (缓存日期)
   ├─ 选项：
   │  1. 使用本地缓存 (快速开始)
   │  2. 更新缓存 (拉取最新)
   └─ 继续到步骤 1

   情况 B：缓存不存在
   ├─ 显示：未找到知识缓存
   ├─ 选项：
   │  1. 初始化缓存 (推荐，首次拉取)
   │  2. 跳过缓存 (直接开始学习)
   └─ 继续到步骤 1
```

## 注意事项

1. **用户确认**：修改多个文件前，总是先显示预览并等待用户确认
2. **日期格式**：统一使用 YYYY-MM-DD 格式
3. **文件路径**：所有路径相对于仓库根目录
4. **错误处理**：如果文件不存在或模块未找到，给出明确的错误提示
5. **进度计算**：基于实际勾选的 checkbox 数量，不要估算
6. **书签状态**：自动管理，用户通过命令交互，不手动编辑文件
7. **缓存策略**：每次询问用户，而非自动判断过期，给予用户完全控制
8. **内容获取优化**：拉取网页内容时使用 `retain_images=false` 跳过媒体文件，提升加载速度
