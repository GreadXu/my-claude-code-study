# 星火的心跳检查任务

## 学习进度检查（每日 2-3 次）

- [ ] 检查 PROGRESS.md，了解当前进度状态
- [ ] 查看 LEARNING_BOOKMARKS.md，提醒继续未完成的书签
- [ ] 检查是否有新的模块需要添加或更新

## 学习记录整理（每日 1 次）

- [ ] 检查 memory/YYYY-MM-DD.md，整理当日学习记录
- [ ] 更新 MEMORY.md，将重要学习心得归档

## Git 同步提醒（学习完成后）

- [ ] 学习完成后，提示用户提交和推送代码
- [ ] 检查 git status，确认工作区状态

## 注意事项

- **频率控制**: 每天检查 2-3 次即可，不要过于频繁
- **时机选择**: 避免深夜（23:00-08:00）打扰用户
- **静默处理**: 如果没有需要关注的，回复 HEARTBEAT_OK

## 何时保持静默（HEARTBEAT_OK）

- 用户刚刚结束学习（< 30 分钟）
- 学习进度没有变化
- 用户明确表示在学习中，不要打扰
- 深夜时间

## 何时主动汇报

- 发现进度异常（长时间未更新）
- 有新的书签需要提醒继续
- Git 工作区有未提交的更改
- 学习达到重要里程碑
